Шаблон
   Включает: место тод 1 страницу, формы, контролы.
   храниться в папке template в модуле в сайте, не преобразуется в страницу, может быть хеширован как html и json
Страница
   Включает: формы, контролы.
   храниться в папке page в модуле в сйате, преобразуется в страницу с вставкой в установленный шаблон(get запрос), и без шаблона(post запрос). кешированным видом является конечный файл в корне сайта(get) как html или файл в модуле в сайте в папк хеша(post) как html или json.
Форма
   Включает: контролы.
   храниться в папке form в модуле сайта, не преобазуется в страницу, может быть хеширована как html или json
Контрол
   Включает: html опредление.
   хранитсья в папке controls в модуле морфера(стандартные) и в модуле страницы(пользовательские). не преобразуется в страницу, кешируется временно парсером в памяти для ускорения разбора файлов.
Общее
   регулярными выражениями в 'body' можно подставлять только параметры из текущего файла, контролы и вормы задаются в параметрах по именам и с входными параметрами и т.д.
   входные параметры указываются в форме или контроле как инпут парм.

Парсер
   на вход передается имя страницы. В зависимости от типа запроса включается в щаблон или нет. Далее строятся формы потом все контролы страницы и форм
   - нет вложенных параметров
   - "body" содержит html и ссылки на параметры
   - в веб интерфейсе "body" можно редактировать как текст(html), через дебагер браузера и с помощью средств wm
   - справочник глобальных параметров
   - в веб интер. контролы и формы управляются как один элемент, прочие html элементы управляются каждый по отдельности
   - контролы и формы задаются как ссылки на параметры с установленными в них параметрами контрола или формы

   {{key}} - пользовательский параметр, ищется в файле по ключу "key"
   {$key$} - системнвый параметр с ключем "key" заменяется системой
      {$coreScript$} - спец. параметр для шаблонов, подключает js платформы
      {$pageScript$} - спец. параметр для шаблонов, подключает js страницы
      {$page$} - спец. параметр для шаблонов, замещается стриницей сайта в парсере
   {#key#} - глобальный параметр, задаются в 1 фале и действуют на всех шаблонах страницах и контролах

id элементов html
   первая цифра указывает тип элемента:
      0 - страница, построена на клиенте
      1 - страница, построена на сервере
      2 - контрол,  построен на клиенте
      3 - контрол,  построен на сервере
   остальные цифры порядковый номер элемента, полеченный при парсинге
